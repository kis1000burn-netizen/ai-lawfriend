# AI법친 7.1-B — 보완 요청 화면 정의서

## 1. 문서 목적

AI법친 7.1-B 보완 요청 화면 정의서는 보완 요청 도메인의 변호사, 의뢰인, 관리자 화면 구조와 화면별 권한, 상태별 노출 기준, 주요 액션, 고지문, QA 기준을 고정하기 위한 기준문서다.

이 문서는 화면 구현 문서가 아니다.  
이 문서는 이후 실제 React 화면 구현 전, 화면 구조와 사용자 흐름을 잠그기 위한 문서다.

## 2. 현재 전제

- AI법친 6.x 운영 Lock 유지
- featureFreeze: true
- 7.1-B supplement migration 적용 완료
- 보완 요청 워크플로우 정의서 완료
- 보완 요청 상태값 정의서 완료
- 보완 요청 데이터 구조 정의서 완료
- 보완 요청 API 명세 정의서 완료
- 보완 요청 구현 정의서 완료
- 보완 요청 service / repository / API route skeleton 완료
- 보완 요청 상태전이 / 권한 guard 정합성 보정 완료
- 이번 단계는 화면 정의 단계이며 실제 화면 구현은 하지 않는다.

## 3. 화면 설계 원칙

1. 6.x 사건 패키지 화면을 직접 변경하지 않는다.
2. 7.1-B 보완 요청 화면은 별도 탭 또는 별도 섹션으로 분리한다.
3. 변호사 화면은 보완 요청 생성, 발송, 재검토, 수용, 재요청 중심으로 구성한다.
4. 의뢰인 화면은 요청 확인, 답변 입력, 첨부 제출, 제출 완료 확인 중심으로 구성한다.
5. 관리자 화면은 전체 현황, 상태 필터, 장기 미응답, 감사 로그 확인 중심으로 구성한다.
6. AI는 후보 문안 또는 요약 보조까지만 표시한다.
7. 최종 판단, 수용, 재요청, 취소는 권한 있는 인간 사용자가 수행한다.
8. 민감정보 원문, accessToken, optionalPin, hash, 첨부파일 직접 URL은 화면에 노출하지 않는다.
9. 상태 배지는 상태값 정의서의 canonical 상태명을 기준으로 한다.
10. 화면 구현 전 QA 체크리스트를 먼저 작성한다.

## 5. 변호사 화면 정의

### 5.1 SCR-SUP-01 변호사 사건 보완 요청 탭

#### 목적

변호사가 사건 패키지 검토 중 해당 사건의 보완 요청 목록과 상태를 확인한다.

#### 주요 표시 항목

- 사건 번호 / 사건명
- 사건 패키지 publicCode 또는 비식별 공유 정보
- 보완 요청 목록
- 보완 요청 상태 배지
- 요청 유형
- 응답 기한
- 의뢰인 확인 여부
- 의뢰인 응답 여부
- 마지막 상태 변경 시각
- 새 보완 요청 작성 버튼

#### 주요 액션

| 액션 | 허용 주체 | 조건 |
|---|---|---|
| 보완 요청 목록 조회 | LAWYER / ADMIN / SUPER_ADMIN | 사건 접근 권한 있음 |
| 새 보완 요청 작성 | LAWYER / ADMIN / SUPER_ADMIN | 사건 접근 권한 있음 |
| 보완 요청 상세 보기 | LAWYER / ADMIN / SUPER_ADMIN | 해당 사건 요청 |
| 응답 재검토 | LAWYER / ADMIN / SUPER_ADMIN | CLIENT_RESPONDED 상태 |

#### 금지

- 의뢰인이 이 화면에서 보완 요청을 생성할 수 없음
- AI가 자동으로 요청을 발송할 수 없음
- 상태값을 임의로 건너뛰는 전이 금지

### 5.2 SCR-SUP-02 보완 요청 작성 화면

#### 목적

변호사가 의뢰인에게 보낼 보완 요청을 작성한다.

#### 입력 항목

| 필드 | 설명 | 필수 |
|---|---|---:|
| requestType | 보완 요청 유형 | 예 |
| title | 요청 제목 | 예 |
| description | 요청 상세 내용 | 예 |
| dueAt | 응답 기한 | 아니오 |
| items | 세부 요청 항목 | 예 |
| allowAttachment | 첨부 허용 여부 | 아니오 |
| noticeAccepted | 고지 확인 | 예 |

#### 상태 흐름

```text
작성 시작 → DRAFT → 발송 → SENT
```

#### 주요 액션

- 임시 저장
- 발송
- 취소
- AI 후보 문안 보기
- 고지문 확인

#### 고지문

> 보완 요청은 변호사의 사건 검토를 위한 추가 정보 요청입니다.  
> AI가 법률 판단을 확정하지 않으며,  
> 최종 검토와 요청 발송은 변호사 또는 권한 있는 사용자가 수행합니다.

### 5.3 SCR-SUP-03 보완 요청 상세 / 재검토 화면

#### 목적

변호사가 의뢰인의 응답을 확인하고, 수용 또는 추가 보완 요청 여부를 결정한다.

#### 주요 표시 항목

- 요청 제목
- 요청 유형
- 요청 상세
- 상태 배지
- 의뢰인 응답 내용
- 첨부자료 목록
- 상태 로그
- 감사 로그 요약
- 재요청 회차
- 검토 메모 후보

#### 주요 액션

| 액션 | 허용 상태 | 결과 상태 |
|---|---|---|
| 재검토 시작 | CLIENT_RESPONDED | UNDER_REVIEW |
| 보완 수용 | UNDER_REVIEW | ACCEPTED |
| 추가 보완 필요 | UNDER_REVIEW | NEEDS_MORE_INFO |
| 재발송 | NEEDS_MORE_INFO | SENT |
| 종료 | ACCEPTED | CLOSED |
| 취소 | DRAFT / SENT | CANCELLED |

#### 금지 액션

- SENT → UNDER_REVIEW 직접 전이
- CLIENT_RESPONDED → ACCEPTED 직접 전이
- CLOSED / CANCELLED / EXPIRED 상태에서 재발송
- 변호사가 의뢰인 응답 route로 응답 등록

## 6. 의뢰인 화면 정의

### 6.1 SCR-SUP-04 의뢰인 보완 요청 목록

#### 목적

의뢰인이 본인 사건에 대해 받은 보완 요청 목록을 확인한다.

#### 표시 항목

- 요청 제목
- 요청 유형
- 상태 배지
- 요청 변호사
- 요청일
- 응답 기한
- 읽음 여부
- 응답 여부
- 제출 완료 여부

#### 주요 액션

- 보완 요청 상세 보기
- 응답 입력
- 제출 완료 확인

#### 금지

- 의뢰인은 보완 요청을 생성할 수 없음
- 다른 의뢰인의 요청을 볼 수 없음
- 만료/취소/종료된 요청에 응답할 수 없음

### 6.2 SCR-SUP-05 의뢰인 보완 요청 상세 / 응답 입력

#### 목적

의뢰인이 변호사의 보완 요청을 확인하고 답변 또는 첨부자료를 제출한다.

#### 표시 항목

- 요청 제목
- 요청 상세
- 요청 유형
- 세부 항목
- 응답 기한
- 제출 전 고지문
- 답변 입력창
- 첨부 업로드 영역
- 제출 버튼

#### 상태 흐름

```text
SENT → CLIENT_VIEWED → CLIENT_RESPONDED
```

#### 입력 항목

| 필드 | 설명 | 필수 |
|---|---|---:|
| responseText | 답변 내용 | 조건부 |
| attachments | 첨부자료 | 조건부 |
| noticeAccepted | 제출 전 고지 확인 | 예 |

답변 내용과 첨부자료 중 하나 이상은 필요하다.

#### 고지문

> 제출하는 답변과 자료는 사건 검토 목적으로 사용됩니다.  
> 민감정보가 포함된 자료를 제출하기 전 내용을 확인해 주세요.  
> AI가 법률 판단을 확정하지 않으며,  
> 최종 검토는 변호사가 수행합니다.

### 6.3 SCR-SUP-06 의뢰인 보완 제출 완료 화면

#### 목적

의뢰인이 보완 응답 제출이 정상 완료되었음을 확인한다.

#### 표시 항목

- 제출 완료 메시지
- 제출 시각
- 제출한 답변 요약
- 첨부자료 개수
- 다음 단계 안내
- 사건 상세로 돌아가기 버튼

#### 다음 단계 안내

> 제출한 보완 내용은 변호사가 재검토합니다.  
> 추가 확인이 필요한 경우 다시 보완 요청이 발송될 수 있습니다.

## 7. 관리자 화면 정의

### 7.1 SCR-SUP-07 관리자 보완 요청 현황

#### 목적

관리자가 전체 보완 요청 상태를 모니터링한다.

#### 표시 항목

- 전체 보완 요청 수
- 상태별 건수
- 장기 미응답 요청
- 만료 예정 요청
- 취소된 요청
- 최근 응답 요청
- 사건별 / 변호사별 필터
- 상태 필터
- 기간 필터

#### 주요 액션

- 목록 조회
- 상태 필터
- 사건 상세 이동
- 보완 요청 상세 보기
- 장기 미응답 확인

#### 제한

관리자는 운영상 확인할 수 있으나, 법률 검토 내용의 주체가 되지 않는다.

### 7.2 SCR-SUP-08 관리자 보완 요청 상세 / 로그

#### 목적

관리자가 특정 보완 요청의 상태 이력과 감사 로그를 확인한다.

#### 표시 항목

- 요청 기본 정보
- 사건 정보
- 요청자 / 대상자
- 상태 로그
- 감사 로그
- 접근 이력
- 취소 / 만료 여부
- 민감정보 마스킹 상태

#### 주요 액션

- 상태 로그 확인
- 감사 로그 확인
- 운영 메모 후보
- 사건 상세 이동

#### 금지

- 첨부파일 직접 URL 노출
- accessToken / optionalPin / hash 노출
- AI raw response 전체 노출
- 민감 사건자료 전문 로그 노출

## 11. AI 표시 기준

AI는 다음 UI 영역에서만 제한적으로 표시할 수 있다.

| 영역 | AI 가능 역할 |
|---|---|
| 보완 요청 작성 화면 | 후보 문안 추천 |
| 보완 요청 상세 화면 | 누락 정보 후보 표시 |
| 의뢰인 응답 확인 화면 | 응답 요약 보조 |
| 재요청 판단 보조 | 추가 질문 후보 제시 |

AI 금지 표시:

- "법률적으로 반드시 이렇다"
- "승소 가능성이 높다"
- "무조건 고소해야 한다"
- "수사기관에 이렇게 대응하면 된다"
- "이 자료는 결정적 증거다"
- 변호사 검토 전 확정 법률 판단
