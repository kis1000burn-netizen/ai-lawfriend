## 1. 문서 목적

AI법친 6.1은 6.0에서 정의한 사건 패키지 / 고유번호 공유 / 변호사 열람 연계 구조를 실제 구현 전에 데이터 기준으로 세분화하는 문서다.

이번 단계의 목적은 다음과 같다.

1. 사건 패키지가 어떤 데이터를 포함하는지 정의한다.
2. 기존 Case / Attachment / LegalDocument / Interview / Summary 데이터를 어떻게 묶을지 정한다.
3. 사건 패키지 생성 가능 조건을 정한다.
4. 공유 가능 데이터와 공유 제한 데이터를 분리한다.
5. 변호사 열람용 패키지 화면에 들어갈 데이터 구성을 확정한다.
6. PDF / 다운로드 / 첨부자료 제공 기준을 정한다.
7. 후속 6.2 고유번호 발급 및 공유 동의 구조의 기준 데이터를 확정한다.

이번 6.1은 실제 DB 변경이나 API 구현이 아니라 데이터 구조 설계 단계다.

## 3. 기본 원칙

사건 패키지는 법률 판단 결과물이 아니다.

사건 패키지는 의뢰인이 입력한 사실관계, AI 인터뷰 결과, 첨부자료, 사건 요약, 문서 초안의 기초를 변호사가 사전 검토할 수 있도록 정리한 자료 묶음이다.

### 핵심 원칙

1. 사건 패키지는 기존 Case를 기반으로 생성한다.
2. 사건 패키지는 의뢰인 소유 사건에서만 생성한다.
3. 사건 패키지는 변호사 검토용 자료 묶음이다.
4. 사건 패키지는 법률 자문이나 최종 문서가 아니다.
5. 변호사 공유 전에는 의뢰인 동의가 필요하다.
6. 변호사 열람 범위는 공유 설정에 따라 제한된다.
7. 첨부파일 다운로드는 별도 권한으로 분리한다.
8. 열람·다운로드는 로그를 남기는 구조로 설계한다.
9. 대시보드 3.x 봉인은 재오픈하지 않는다.
10. 기존 CaseStatus canonical source는 변경하지 않는다.

## 4. 사건 패키지 개념

사건 패키지란, 하나의 사건 Case를 기준으로 의뢰인 입력 정보, AI 인터뷰 답변, 사건 요약, 첨부자료, 문서 초안의 기초, 추가 확인 필요 항목을 변호사 검토용으로 묶은 구조화 자료다.

### 기능명

기능명:
AI법친 사건 패키지

### 영문 후보

- AI Lawfriend Case Package
- LawCase Brief Package
- Case Review Package

### 권장 표시명

의뢰인 화면:
내 사건 패키지

변호사 화면:
사건 패키지 조회

관리자 화면:
사건 패키지 공유 관리

## 5. 사건 패키지 데이터 구성

사건 패키지는 아래 8개 영역으로 구성한다.

1. 패키지 기본 정보
2. 사건 기본 정보
3. 의뢰인 / 상대방 정보
4. AI 인터뷰 / 답변 정보
5. 사건 요약 / 쟁점 후보
6. 첨부자료 정보
7. 문서 초안 / 문서 상태 정보
8. 공유 / 열람 / 다운로드 제어 정보

## 6. 패키지 기본 정보

패키지 기본 정보는 사건 패키지 자체를 식별하기 위한 정보다.

| 필드 | 설명 | 공유 여부 |
| --- | --- | --- |
| `packageId` | 내부 패키지 ID | 내부용 |
| `caseId` | 연결된 사건 ID | 내부용 |
| `publicCode` | 변호사 조회용 고유번호 | 공유 가능 |
| `packageTitle` | 사건 패키지 제목 | 공유 가능 |
| `packageStatus` | `DRAFT / READY / SHARED / EXPIRED / REVOKED` | 제한 공유 |
| `createdAt` | 패키지 생성일 | 공유 가능 |
| `updatedAt` | 패키지 갱신일 | 공유 가능 |
| `expiresAt` | 공유 만료일 | 공유 가능 |
| `ownerUserId` | 의뢰인 사용자 ID | 내부용 |
| `sharedLawyerUserId` | 지정 변호사 ID | 내부용 |

### packageStatus 후보

`DRAFT`:
패키지 생성 전 또는 작성 중

`READY`:
공유 가능 상태

`SHARED`:
변호사에게 공유 중

`EXPIRED`:
공유 기간 만료

`REVOKED`:
의뢰인이 공유 취소

`ARCHIVED`:
보관 처리

## 7. 사건 기본 정보

기존 Case 데이터를 기반으로 구성한다.

| 필드 | 설명 | 변호사 공유 |
| --- | --- | --- |
| `caseTitle` | 사건 제목 | 가능 |
| `caseType` | 사건 유형 | 가능 |
| `caseStatus` | 사건 진행 상태 | 가능 |
| `createdAt` | 사건 생성일 | 가능 |
| `updatedAt` | 사건 최종 수정일 | 가능 |
| `incidentDate` | 사건 발생일 | 가능 |
| `claimAmount` | 피해금액 / 청구금액 | 가능 |
| `summaryStatus` | 요약 생성 여부 | 가능 |
| `documentStatus` | 문서 초안 상태 | 가능 |

단, 내부 운영 상태값을 그대로 노출하면 안 되는 경우가 있다.

예:

- 관리자 내부 상태
- 검증용 상태
- 삭제 / 보류 / 운영 이슈 상태
- 내부 오류 상태

변호사 열람 화면에는 사람이 이해하기 쉬운 표시값으로 변환한다.

예:

- `IN_INTERVIEW` → 인터뷰 진행 중
- `INTERVIEW_DONE` → 인터뷰 완료
- `REVIEW_PENDING` → 검토 대기
- `APPROVED` → 승인 완료

## 13. 사건 패키지 생성 조건

사건 패키지는 언제든 생성할 수 있으면 안 된다. 최소 조건이 필요하다.

### 최소 생성 조건

1. 사건이 존재해야 한다.
2. 사건 소유자가 현재 사용자여야 한다.
3. 사건이 삭제 상태가 아니어야 한다.
4. 기본 사건 정보가 있어야 한다.
5. 의뢰인이 공유 목적과 동의 항목을 확인해야 한다.

### 권장 생성 조건

1. AI 인터뷰가 완료되었거나 일부 답변이 존재한다.
2. 사건 요약이 생성되어 있다.
3. 첨부자료 목록이 정리되어 있다.
4. 문서 초안의 기초가 있거나 생성 가능하다.
5. 추가 확인 필요 항목이 정리되어 있다.

### 생성 제한 조건

아래 상태에서는 패키지 공유를 제한한다.

- 사건 삭제됨
- 의뢰인 본인 사건 아님
- 공유 동의 없음
- 필수 개인정보 고지 미확인
- 내부 운영 보류 상태
- 법률상 공유 제한이 필요한 자료 포함

## 14. 사건 패키지 생성 시점

추천 생성 시점은 3가지다.

1. AI 인터뷰 완료 후
2. 사건 요약 생성 후
3. 의뢰인이 변호사에게 공유하기 버튼을 누른 시점

최종 추천은 3번이다.

사건 패키지는 자동 생성하지 않고,
의뢰인이 변호사에게 공유하기를 선택할 때 생성한다.

이유:

- 의뢰인 동의가 명확하다.
- 공유 범위를 직접 선택할 수 있다.
- 불필요한 패키지 생성을 줄인다.
- 개인정보 제공 근거가 명확해진다.

## 17. 사건 패키지 데이터 스냅샷 기준

사건 패키지는 항상 최신 Case를 실시간으로 보여줄지, 공유 시점 스냅샷을 남길지를 정해야 한다.

추천은 혼합형이다.

기본 정보:
실시간 조회

공유 동의 / 공유 범위:
스냅샷 저장

공유 당시 요약:
스냅샷 저장

첨부파일:
권한 기준으로 실시간 조회

다운로드 권한:
스냅샷 + 현재 상태 모두 확인

공유 취소 / 만료:
현재 상태 우선

### 이유

- 변호사가 본 내용의 기준 시점을 추적할 수 있다.
- 의뢰인이 나중에 내용을 수정해도 공유 당시 기준을 확인할 수 있다.
- 첨부파일 삭제·취소·만료 같은 보안 상태는 실시간 반영되어야 한다.

## 20. 신규 API 후보

6.1은 설계 단계이므로 실제 구현하지 않는다.

- `GET    /api/cases/[caseId]/package-preview`
- `POST   /api/cases/[caseId]/package`
- `GET    /api/cases/[caseId]/package`
- `PATCH  /api/cases/[caseId]/package`

### 각 API 역할

`GET package-preview`:
현재 사건 데이터로 패키지 미리보기 생성

`POST package`:
의뢰인 동의 후 사건 패키지 생성

`GET package`:
생성된 패키지 조회

`PATCH package`:
패키지 제목 또는 공유 전 범위 수정

공유 API는 6.2에서 분리한다.

## 22. 개인정보 / 보안 기준

1. 사건 패키지는 의뢰인 본인 사건만 생성 가능하다.
2. 변호사 공유 전 명시적 동의가 필요하다.
3. 첨부파일 다운로드는 별도 권한이 필요하다.
4. 민감정보는 기본 비공유 또는 마스킹한다.
5. 내부 운영 정보는 절대 공유하지 않는다.
6. 공유 취소 시 즉시 접근을 차단한다.
7. 만료 시 접근을 차단한다.
8. 열람 / 다운로드 로그를 남길 수 있도록 설계한다.
