# Voice Transcript QA (VOICE_TRANSCRIPT_QA)

**상태**: Phase **5-A** 초안 · STT 불확실성 대응·운영 QA 체크리스트 SSOT  

**증빙 태그**: `[EVIDENCE-20260523-AIBEOPCHIN-PHASE5A-VOICE-INTAKE-INTERVIEW-LAYER]`

## 1. 왜 필요한가

법률 맥락에서 잘못된 STT 한 글자·한 숫자가 **증거 평가·청구 금액 오인**으로 이어질 수 있다. 따라서 QA의 핵은 **모델 신뢰도가 높더라도** “사용자 확인 전 확정 금지”가 아니라, **실제 사람이 무엇을 보고 서명했는지**를 입증 가능하게 만드는 것이다.

## 2. 사용자 대면 원칙 (UX·복지)

확정 화면(`NEEDS_CONFIRMATION`)에서 다음을 만족시킨다.

1. STT 결과를 **항상 수정 가능**(읽기 전용 노출 후 비활성 금지).
2. “이 내용으로 인터뷰 답변에 반영합니다” **명시 버튼** — 암묵 탭 저장 금지.
3. **STT 불확실성 고정 문구**(짧게): 오인식 가능, 불일치 시 직접 수정 권유.
4. 숫자·날짜·고유명사·주소에는 **추가 강조**(하이라이트·재확인 질문 5-E와 연계).
5. `REJECTED` 시 **어떤 데이터도 반영 안 됨**을 한 줄로 명시.

## 3. 테스트 시나리오 (향후 확장 검증은 **Phase 5‑H** 로드맵까지 이관 가능; 여기 초안 명시 유지)

| ID | 시나리오 | 기대 |
|----|----------|------|
| QA-V-01 | STT 성공 → 확인 없이 새로고침 | 초안 유지 또는 만료 정책에 따름, **인터뷰 무갱신** |
| QA-V-02 | STT 빈 문자열·저신뢰 플래그 | `NEEDS_CONFIRMATION`에서 사용자 안내·재발화 유도 |
| QA-V-03 | 수정 후 확인 | 수정본만 `CONFIRMED`·`VOICE_INTERVIEW_ANSWER_BOUND` |
| QA-V-04 | 거절 | `REJECTED`, 답변 필드 무변 |
| QA-V-05 | 권한 없는 사용자(타인 사건 URL·ID) | 음성·transcript 접근 차단(API 403 등) |

## 4. 변호사 검토 전 차단 (**Phase 5‑H** 명세 SSO)

근거 명세: [`VOICE_LAWYER_REVIEW_UX_SPEC.md`](./VOICE_LAWYER_REVIEW_UX_SPEC.md).

아래 구간에서는 **법률 문서 최종 출력·외부 발송 채널**과 논리 분리된다.

- `CONFIRMED` transcript만 인터뷰 답변으로 반영.
- 문서 초안 생성은 기존 **문서 안전·변호사 워크플로** 규약을 따른다 ([`GONGBUHO_SAFETY_POLICY.md`](../gongbuho/GONGBUHO_SAFETY_POLICY.md) 정신 동일선).

운영 증명용으로 **`VOICE_*` 추적 이벤트 타임라인을 사건 카드 또는 백오피스에서 조회 가능**하게 두는 것을 권장한다.

## 5. 메트릭(선택, 제품 허용 시)

민감 식별자 없이 통계 가능한 항목만:

- STT 처리 실패율
- 사용자 편집 거리(edit distance 분포 또는 구간별 버킷)
- `CONFIRMED` 평균 시간

법률·개인정보 검토 후 수집.
