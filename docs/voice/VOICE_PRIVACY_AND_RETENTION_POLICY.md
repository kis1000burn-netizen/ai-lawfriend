# Voice Privacy · Retention (VOICE_PRIVACY_AND_RETENTION_POLICY)

**상태**: Phase **5-A** 초안 · 개인정보·보존 최소값 SSOT · 법무 확정 필요

**증빙 태그**: `[EVIDENCE-20260523-AIBEOPCHIN-PHASE5A-VOICE-INTAKE-INTERVIEW-LAYER]`

## 1. 목적

음성 데이터는 **텍스트보다 민감도가 높고 복제·재식별 위험**이 있다. 본 문서는 **원본 음성·중간 산출물·확정 transcript**의 수집·보관·삭제·접근을 최소화하는 **기본 제품 방향**을 정한다.

## 2. 데이터 분류

| 유형 | 예시 | 민감도 |
|------|------|--------|
| **원본 음성** | 웹m/오디오 blob, 스트림 청크 | 높음 |
| **STT 원시 JSON** | 타임스탬프, 대안 hypothesis | 중~높음 |
| **Draft transcript** | `NEEDS_CONFIRMATION` 텍스트 | 중 |
| **Confirmed transcript** | `CONFIRMED` 후 인터뷰에 반영된 문자열 | 기존 인터뷰 데이터와 동일 등급 |

## 3. 기본 정책 (권장 기본값)

1. **원본 음성**: 기본 **저장하지 않음**. STT 요청 직후 버퍼에서 파기.
2. **선택 저장**: 명시 옵트인(법무·약관 문구)일 때만 암호화 저장, **짧은 TTL** 또는 사건 종결 시 삭제 루틴.
3. **Draft transcript**: 확인 전 단계에서만 보관, `REJECTED` 또는 만료 시 삭제(정책 TTL).
4. **Confirmed**: 인터뷰 답변과 동일한 **사건 접근 통제** — STAFF/변호사 조회는 기존 RBAC에 따름.
5. **로그**: 음성 전문·전체 transcript를 **애플리케이션 로그에 평문 기록 금지**.

## 4. 접근 통제

- **본인(의뢰인)**: 본인 사건의 draft/confirmed 조회·수정·삭제(정책 허용 범위).
- **담당 변호사·승인된 스태프**: 기존 사건 읽기 권한과 정렬.
- **운영**: 분쟁 대응 최소 범위, **목적 외 열람 금지** 내부 규정 링크.

## 5. 제3자 처리(STT/TTS 공급자)

- **DPA·지역**은 별도 체크리스트(**5‑I** Privacy 감사 락).
- 가능하면 **일시 처리 + 로그 최소화** 계약 옵션 우선.
- 사용자 고지: 음성이 **외부 엔진으로 전송될 수 있음**(해당 시).

## 6. 보존 기간(가이드 — 법무 확정 전)

| 데이터 | 권장 기본 |
|--------|-----------|
| 원본 음성(옵트인) | 최대 N일 또는 STT 완료 후 즉시 — **N은 법무 합의** |
| Draft | 세션 또는 24~72h 등 **짧은 TTL** 후 파기 |
| Confirmed / 인터뷰 답변 | 기존 사건 데이터 보존 정책과 **동일 선상**에서 관리 |
