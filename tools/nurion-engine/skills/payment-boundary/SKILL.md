# payment-boundary 스킬

## 목적
P-0 결제 경계를 보호합니다.  
고객 폼의 금액 입력·전송 검사, 관리자 권한 분리, 결제 상태 전환을 검증하며  
결제 관련 AI 자동 실행은 **일절 금지**합니다.

## 언제 사용하는가
- 결제·견적 금액이 포함된 모든 플랫폼  
- `save_payment` / `confirm_payment` / `update_price` 함수가 존재할 때  
- 정기 감시 사이클마다 P-0 경계 유지 확인

## 판단 기준
| 조건 | 결과 |
|------|------|
| `code:p0-boundary` 신호가 fail | FAIL — critical (G4 트리거) |
| 결제 엔드포인트에 인증 헤더 없이 200 응답 | FAIL — critical |
| `payment_waiting` 상태가 probe에 없음 | WARN |
| 결제 금액 필드가 클라이언트에서 직접 전송 가능 | WARN |

## 자동조치 (actions.mjs)
| 조치 | 허용 여부 |
|------|-----------|
| 접수 차단 알림(notify + escalate) | ✅ 허용 |
| `PAYMENT_BLOCK=1` 플래그 전송 (G3 이상) | ✅ apply 모드에서 허용 |
| 결제 금액 변경 | ❌ **절대 금지** |
| 환불 처리 | ❌ **절대 금지** |
| 가격 수정 | ❌ **절대 금지** |
| 결제 승인·취소 API 호출 | ❌ **절대 금지** |

## 안전 정책 (policy.json 참조)
결제 관련 조치는 **사람 승인 없이 AI가 실행하는 것이 금지**됩니다.  
이 스킬은 차단과 알림까지만 허용되며, 그 이상의 조치는 운영자가 직접 수행합니다.
