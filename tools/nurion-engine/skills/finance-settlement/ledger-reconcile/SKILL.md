# ledger-reconcile 스킬

## 목적
주문 → PG 결제 → GAS 기록 → 업체 정산 예정액 → 실제 입금의  
각 단계 금액을 자동 비교(대사)하고 불일치를 즉시 감지합니다.

누리온이 직접 돈을 보내지 않습니다.  
**"한 푼이라도 어긋나면 멈추고 근거를 보여준다"** 가 이 스킬의 역할입니다.

## 대사 공식
```
payout = 주문금액
       - 할인
       - 취소·환불
       - PG 수수료
       - 배송비 (vendor_borne 인 경우)
       - 플랫폼 수수료
       = 업체 지급 예정액
```

## 언제 사용하는가
- 정산 주기(주간/월간) 도래 시  
- PG 정산 파일 수신 후  
- 업체 이의 제기 발생 시  
- 환불·취소 처리 후 재대사

## 판단 기준
| 조건 | 신호 | 등급 |
|------|------|------|
| 예상 지급액 ≠ 실제 지급액 (1원 이상) | `finance:payout-mismatch` | G3 |
| 정산 데이터 없음 | warn | — |
| 미대사 건수 > 임계값 | warn | G2 |

## 자동조치 (actions.mjs)
| 조치 | 허용 여부 |
|------|-----------|
| `SETTLEMENT_HOLD` 플래그 전송 | ✅ apply + G3 이상에서 허용 |
| `SETTLEMENT_MISMATCH` 인시던트 생성·저장 | ✅ 항상 허용 |
| 알림·에스컬레이션 | ✅ 항상 허용 |
| 지급 금액 변경 | ❌ **절대 금지** |
| 송금 실행 | ❌ **절대 금지** |
| 원본 거래 수정 | ❌ **절대 금지** |
